Prepare Slides

If you're a Red Hatter, a template for the presentation that originally accompanied this demo is available here.

Configure Slack

The slack component requires a token that corresponds with your user account. You can view or create tokens here. Once you have one for the Red Hat workspace, you need to provide it in the SLACK_TOKEN environment variable for the slack2node deployment configuration in OpenShift. If you wish to override the default channels used for Red Team and White Team inputs, you can modify the RED_CHANNEL_ID and WHITE_CHANNEL_ID here as well.

Create a Shortened URL

Creating a shortened URL makes it easier for participants to join. You'll need to direct them to the route created for the landing-page deployment. If you're a Red Hatter, you can use the Red Hat URL Shortener (requires VPN), or otherwise tinyURL or bit.ly. You'll need to update the join-link ConfigMap and then redeploy directive-consumer to show the shortened URL on the game's page.

Create a QR Code

A QR Code allows participants to join the game simply by pointing their camera at the screen. We used a free QR code generator, but you should be cautious of the scan limits depending on the size of your audience.

Running the Demo

Access the game at the route for directive-consumer. As soon as you click Play, the game will start and begin collecting inputs. Make sure you're ready! Once the game is complete, metrics will display on the screen. The MVP on each team is the person who agreed with the consensus the most. The Biggest Troll on each team is the person who disagreed with the consensus the most.
